第 3 章 · 学 Paimon

Paimon 是什么

地基打好了，现在正式认识 Paimon。这一章用大白话讲清它的定位、核心特性和典型场景。

一句话定位

官方定位

Apache Paimon 是一个数据湖表格式（lake format），配合 Flink 和 Spark，支持流处理和批处理，用来构建实时湖仓架构（Realtime Lakehouse）。

它的前身是 Flink Table Store，现在是 Apache 顶级项目。

用更通俗的话说

对零基础来说，可以先这样理解：

Paimon 是一个给大数据系统用的「数据底座」，专门处理「数据不断变化、还要实时查和分析」的场景。

它既不是单纯的数据库，也不是单纯的消息队列，而是让流 + 批 + 数据湖存储更统一的一层。

核心特性

流批一体
同一份数据，既能流式消费，也能批量分析，不用维护两套存储。
主键更新强
基于 LSM 思路处理更新，特别适合高频 upsert 和 CDC 入湖。
Changelog
能产出数据变化记录，供下游做增量计算。
Time Travel
可以查询历史某个时间点 / 版本的数据，便于回溯和排查。
Schema Evolution
表结构变了（加字段等）也不用重写历史数据。
Flink 原生
和 Flink 集成最强，也支持 Spark、Trino、StarRocks、Doris 等查询。

典型使用场景

  • CDC 实时入湖：MySQL/PostgreSQL 的变更实时同步进数据湖。
  • 实时宽表 / 实时特征表：多张表实时拼成一张大宽表供查询。
  • 近实时明细查询：既要明细、又要响应快的 ad-hoc 查询。
  • 流批统一：用一套存储同时支撑实时和离线，降本增效。
现在先不用纠结

LSM、manifest、snapshot、merge engine 这些底层词，下一章会专门讲。这里只要先记住 Paimon 的「定位 + 特性 + 场景」就够了。
