第 1 章 · 打基础

预备知识

这一章是最底层的地基：会用终端、看懂 SQL、理解「表」相关的基本概念。不难，但必须有。

1. 工具基础：Linux 命令

Paimon、Flink、Spark 的教程大多默认你会基本命令行。先掌握这几个就够：

命令作用
ls列出当前目录的文件
cd 目录进入某个目录
pwd显示当前所在路径
cat 文件查看文件内容
grep 关键词 文件在文件里搜索关键词
示例：进入目录并查看文件
cd /tmp/paimon
ls
cat README.txt
目标

能打开终端、切换目录、查看文件、照着教程执行命令。够用即可，不用深入。

2. 数据基础：SQL

这一层比工具更重要。Paimon 本质还是围绕「表」和「数据变化」展开，SQL 是你和数据对话的语言。先学这些：

最该掌握的 SQL
-- 查询：从订单表里取北京、金额大于 100 的订单
SELECT user_id, amount
FROM orders
WHERE city = 'Beijing' AND amount > 100;

-- 聚合：按城市统计订单数和总金额
SELECT city, COUNT(*) AS cnt, SUM(amount) AS total
FROM orders
GROUP BY city
ORDER BY total DESC;

-- 关联：把订单和用户表按 user_id 拼起来
SELECT o.order_id, u.name, o.amount
FROM orders o
JOIN users u ON o.user_id = u.user_id;
目标

看得懂一张表结构，能写简单查询，知道数据怎么被取出、过滤、聚合、关联。

3. 必懂的「表」相关概念

表 / 字段 / 主键

表就像 Excel 的一张工作表，字段是列，每一行是一条记录。主键（Primary Key）是能唯一标识一行的字段，比如订单号——主键相同就认为是同一条记录，这点在 Paimon 的「更新」里非常关键。

分区（Partition）

把一张大表按某个字段（常见是日期）切成多块存放，比如按 dt=20260101 一天一块。查询时只扫需要的分区，速度更快。

ETL

Extract（抽取）、Transform（清洗转换）、Load（加载）。即把数据从源头取出来、清洗加工、再写到目标表里。数据系统每天都在干这件事。

数据仓库（数仓）

专门用来做分析的「大表仓库」。和业务数据库不同，它更偏离线分析、报表、统计。理解数仓，是理解「湖仓（Lakehouse）」的前提。

把这些词记牢

表、字段、行、主键、分区、ETL、数仓——后面讲 Paimon 时会反复用到。尤其是主键和分区，直接关系到 Paimon 为什么擅长更新。
