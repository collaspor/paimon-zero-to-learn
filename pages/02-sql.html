SQL 与数据建模

SQL 是你和数据对话的语言，Paimon 的建表、写入、查询全靠它。这一章带你掌握够用的查询能力，并理解表、主键、分区、ETL、数仓这些核心概念。

表是什么

一张表（Table）就像 Excel 的一张工作表：

  • 列 / 字段（Column）：表的每一项属性，如 order_id、amount。
  • 行 / 记录（Row）：一条具体数据，如「订单 1001，金额 200」。
  • 数据类型：每列有类型，如整数 INT、长整数 BIGINT、字符串 STRING、小数 DECIMAL。
建一张订单表
CREATE TABLE orders (
    order_id   BIGINT,
    user_id    BIGINT,
    city       STRING,
    amount     DECIMAL(10, 2),
    created_at TIMESTAMP
);

核心查询：SELECT

查询是 SQL 用得最多的能力。先掌握这四件事：选列、过滤、聚合、排序。

查询：选列 + 过滤
-- 从订单表取 user_id 和 amount
-- 只要北京、金额大于 100 的订单
SELECT user_id, amount
FROM orders
WHERE city = 'Beijing' AND amount > 100;
聚合：分组统计
-- 按城市统计：订单数、总金额、平均金额
SELECT
    city,
    COUNT(*)     AS order_cnt,
    SUM(amount)  AS total_amount,
    AVG(amount)  AS avg_amount
FROM orders
GROUP BY city
ORDER BY total_amount DESC;

常用聚合函数：COUNT（计数）、SUM（求和）、AVG（平均）、MAX/MIN（最大/最小）。GROUP BY 决定「按什么分组」。

关联：JOIN

当信息分散在多张表，用 JOIN 把它们拼起来：

把订单表和用户表拼起来
-- orders 里只有 user_id，想知道用户名要去 users 表取
SELECT o.order_id, u.name, o.amount
FROM orders o
JOIN users u ON o.user_id = u.user_id;
JOIN 直觉

JOIN 就是「按某个相同的字段（这里是 user_id）把两张表对齐拼接」。在实时数仓里，把多张表 JOIN 成一张「宽表」是非常常见的需求，也是 Paimon 的典型用途之一。

必懂概念

主键 PRIMARY KEYPrimary Key

能唯一标识一行的字段，比如订单号。主键相同就认为是同一条记录——这点在 Paimon 的「更新（upsert）」里至关重要：主键相同的新数据会覆盖旧数据。

分区 PARTITIONPartition

把一张大表按某字段（常见是日期）切成多块存放，比如按 dt=20260101 一天一块。查询时只扫需要的分区，速度更快、成本更低。

ETLExtract-Transform-Load

抽取（从源头取数据）→ 转换（清洗、加工）→ 加载（写入目标表）。数据平台每天都在做这件事，CDC 入湖本质也是一种 ETL。

数据仓库 数仓Data Warehouse

专门用于分析的「大表仓库」。和支撑业务的数据库不同，它面向报表、统计、离线分析。理解数仓是理解「湖仓（Lakehouse）」的前提，下一阶段会讲。

分区表长什么样

带分区的表
-- 按日期 dt 分区，每天的数据单独一块
CREATE TABLE orders_partitioned (
    order_id BIGINT,
    amount   DECIMAL(10,2),
    dt       STRING       -- 分区字段，如 '20260101'
) PARTITIONED BY (dt);

-- 查询只扫某一天，速度快
SELECT SUM(amount) FROM orders_partitioned WHERE dt = '20260101';
练习 2-1

有一张 orders(order_id, city, amount) 表。请写 SQL：统计每个城市金额大于 50 的订单数量，按数量从多到少排序。

查看参考答案
参考答案
SELECT city, COUNT(*) AS cnt
FROM orders
WHERE amount > 50
GROUP BY city
ORDER BY cnt DESC;
