第 5 章 · 学 Paimon

Quick Start 上手

跟着官方思路，用 Flink SQL 跑一个最小示例：建 catalog、建表、流式写入、批查、流查。

前置说明

这一章涉及实际环境，零基础同学第一次看可以先「读懂流程」，等装好 Flink 再动手。版本和 jar 以官方 Quick Start 为准。

步骤总览

  1. 下载 Flink 并解压。
  2. 把 Paimon 的 bundled jar 拷到 Flink 的 lib/ 目录。
  3. （在 Hadoop 环境外）准备好 Hadoop 依赖。
  4. 启动 Flink 本地集群，打开 SQL Client。
  5. 建 catalog 和表 → 写入 → 查询。

1. 准备 Flink 与 Paimon

解压 Flink 并放入 Paimon jar
# 解压 Flink
tar -xzf flink-*.tgz

# 把 Paimon bundled jar 拷到 Flink 的 lib 目录
cp paimon-flink-*.jar <FLINK_HOME>/lib/

# 启动本地集群并打开 SQL Client
<FLINK_HOME>/bin/start-cluster.sh
<FLINK_HOME>/bin/sql-client.sh

启动后可访问 localhost:8081 查看 Flink Dashboard。

2. 创建 Catalog 和表

建 catalog 与主键表
-- 创建一个 Paimon catalog，数据存在本地目录
CREATE CATALOG my_catalog WITH (
    'type' = 'paimon',
    'warehouse' = 'file:/tmp/paimon'
);

USE CATALOG my_catalog;

-- 建一张 word_count 表，word 是主键
CREATE TABLE word_count (
    word STRING PRIMARY KEY NOT ENFORCED,
    cnt BIGINT
);

3. 流式写入数据

用数据生成器持续写入
-- 造一个不断产生单词的数据源
CREATE TEMPORARY TABLE word_table (
    word STRING
) WITH (
    'connector' = 'datagen',
    'fields.word.length' = '1'
);

-- 流模式下 Paimon 需要 checkpoint
SET 'execution.checkpointing.interval' = '10 s';

-- 持续把单词计数写进 word_count
INSERT INTO word_count SELECT word, COUNT(*) FROM word_table GROUP BY word;

4. 批量查询（OLAP）

切到批模式查询当前结果
SET 'sql-client.execution.result-mode' = 'tableau';

-- 切换到批模式
RESET 'execution.checkpointing.interval';
SET 'execution.runtime-mode' = 'batch';

-- 查询当前 word_count（可多次执行，观察变化）
SELECT * FROM word_count;

5. 流式查询

切到流模式持续观察变化
SET 'execution.runtime-mode' = 'streaming';

-- 持续跟踪表的变化
SELECT `interval`, COUNT(*) AS interval_cnt FROM
    (SELECT cnt / 10000 AS `interval` FROM word_count) GROUP BY `interval`;
你刚刚做了什么

同一张 word_count 表，既能流式写入，又能批量查当前快照、还能流式查变化——这就是「流批一体」的直观体验，也是 Paimon 最核心的卖点。

收尾

停止集群
# 在 localhost:8081 取消流作业后，停止本地集群
<FLINK_HOME>/bin/stop-cluster.sh

跑通这个示例，你就完成了从 0 到「亲手用过 Paimon」的关键一步。下一章看它和其他表格式怎么比。
