第 0 章

零基础学习路线

Paimon 对零基础并不友好。最聪明的做法不是直接学它，而是先补一层地基，再进入正题。

为什么不能直接学 Paimon

Paimon 的文档默认你已经懂 SQL、懂表、懂 Flink、懂数据湖。如果这些都不熟，你会卡在「这个词是什么意思」，而不是「这个设计为什么好」。所以我们先打地基。

四层地基 → Paimon

按下面顺序学，每一层都为下一层做铺垫：

第 1 层
工具基础
Linux 命令、文件路径、终端操作。目标：能照着教程执行最小命令。
第 2 层
数据基础
SQL、表、主键、分区、ETL、数仓。目标：理解数据怎么被存、查、加工。
第 3 层
计算基础
批 vs 流、Spark/Flink、CDC。目标：知道 Paimon 为什么常配 Flink。
第 4 层
湖仓概念
snapshot、compaction、time travel。目标：看懂 Paimon 文档的术语。

建议时间安排

阶段学什么建议时长达标标志
第一阶段Linux 基础 + SQL 基础 + 数仓概念约 1 周能写简单查询、看懂表结构
第二阶段批/流区别 + Spark/Flink + CDC约 1–2 周能说清流批差异和 CDC 是什么
第三阶段Paimon 概念 + Quick Start约 3–5 天能跑通最小示例、理解核心特性
第四阶段对标对比 + 实践文章按需能回答「为什么选 Paimon」
现在先别碰

Paimon 源码、复杂 Flink 作业、LSM 底层细节、四种表格式的深度选型对比——这些现在学性价比太低，等地基打好再说。

本周最小目标

  1. 学会基础 SQL 查询（select / where / group by / join）。
  2. 理解「表、主键、分区、ETL」这几个词。
  3. 知道 Spark、Flink、CDC 分别是干什么的。

完成这三件事，再看 Paimon，就不会完全懵。下一章我们就从预备知识开始。
