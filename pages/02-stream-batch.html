计算基础

这一章让你理解 Paimon 所在的世界：什么是批处理和流处理，Spark 和 Flink 各管什么，CDC 又是什么。

批处理 vs 流处理

批处理（Batch）流处理（Streaming）
处理方式攒一批数据，定时跑一次数据来一条处理一条，持续运行
时效一般 T+1（隔天）秒级 / 分钟级
典型场景每日报表、离线统计实时大屏、实时风控、实时入湖
代表引擎Spark（也支持流）Flink（流处理见长）

一句话：批处理像「每天结算一次」，流处理像「随时实时记账」。

Spark 和 Flink 各是什么

Apache Spark

大数据领域最流行的计算引擎之一，批处理和分析见长，生态成熟，常用于离线数仓、机器学习。也支持流处理（Structured Streaming）。

Apache Flink

流处理见长的引擎，强在低延迟、状态管理、Exactly-Once。它是 Paimon 的「最佳搭档」——Paimon 最早就来自 Flink 社区（原名 Flink Table Store）。

CDC 是什么

CDC = Change Data Capture（变更数据捕获）。简单说：把业务数据库（如 MySQL）里的「增、删、改」实时同步出来，喂给下游系统。

为什么 CDC 重要

很多实时数仓的第一步就是：MySQL → CDC → Flink → Paimon。把线上数据库的变化实时灌进数据湖，这正是 Paimon 最擅长的场景之一。

为什么 Paimon 常和 Flink 一起出现

你在 Paimon 资料里会反复看到这样的链路：

典型实时入湖链路
MySQL (业务库)
   │  变更（增删改）
   ▼
CDC 工具 / Flink CDC
   │  实时同步
   ▼
Flink (流计算)
   │  写入
   ▼
Paimon (实时湖仓表)
   │  既能流式消费，也能批量分析
   ▼
Spark / Flink / 查询引擎

看懂这张图，你就理解了 Paimon 的价值：它是这条链路里那个「既能实时写、又能高效查」的存储底座。

目标

能说清批处理和流处理的区别，知道 Flink、Spark、CDC 在链路里各干什么。做到这一步，下一章正式认识 Paimon 就轻松了。
